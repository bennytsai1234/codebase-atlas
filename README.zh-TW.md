# Codebase Atlas

**用於人工智慧輔助開發的代碼庫導航協議與工程架構治理系統**

## 專案願景

Codebase Atlas 是一套專為解決 AI 代理人（AI Agents）在處理大型且複雜代碼庫時常見的「上下文迷失」與「架構漂移」問題而設計的系統化協議。它不單是一個文檔生成工具，更是一個將專案智慧（Project Intelligence）從易失的對話上下文轉化為持久化、可版本化資產的工程方案。

透過這套系統，開發者能為 AI 代理人建立一套完整的「導航層」，使其在動手修改代碼前，能先理解模組邊界、依賴關係、潛在風險以及必須遵循的工程規範。

## 核心價值

### 1. 減少架構幻覺
AI 代理人在缺乏全局視角時，容易做出破壞模組封裝或引入循環依賴的改動。Codebase Atlas 強制 AI 在任務開始前讀取已定義好的「地圖」，確保任何變更都符合既定的設計意圖。

### 2. 智慧持久化與槓桿效應
目前的 AI 工具通常在每次會話中重新探索代碼庫，這造成了顯著的運算資源與 Token 浪費。Codebase Atlas 提倡「一次性深度初始化」的策略，將最強模型的分析結果固化，讓後續的日常任務能以更低的成本獲得高品質的指導。

### 3. 工程治理與工作流標準化
本系統產出的不僅是靜態文檔，還包含針對不同開發情境（如錯誤修復、功能擴展、系統重構）的動態工作流指引，將團隊的工程標準直接嵌入 AI 的執行路徑中。

## 運作模型

Codebase Atlas 採用「初始化一次，多次重複使用」的運作機制：

1.  **地圖構建（Initialization）**：
    在專案啟動或發生重大架構變更時執行。建議使用當前環境中**性能最強的模型**進行此階段的深度掃描與邏輯推理，以確保產出的地圖具備足夠的戰略深度與準確性。

2.  **日常導航（Navigation）**：
    在地圖建立完成後，一般的開發任務不應再次執行 Codebase Atlas。相反地，AI 代理人應被引導讀取產出的 `docs/` 文件，並依照其中的工作流進行作業。這使得較輕量或低成本的模型也能在高品質地圖的輔助下，完成精準的代碼開發。

3.  **地圖重建（Full Rebuild）**：
    僅在使用者明確要求重新掃描、重新整理或重建地圖時，才再次調用本技能進行全量掃描。

## 產出物結構

產出的地圖會存放在專案根目錄的 `docs/` 資料夾下，並遵循嚴格的檔案命名規範：

*   **專案索引 (`_index.md`)**：定義專案全景、交付策略、模組摘要及工作流入口。
*   **模組文檔 (`docs/project/module.md`)**：詳述每個功能模組的當前狀態、邊界範圍、上游依賴與下游影響。
*   **工作流指引 (`_workflow.md`)**：針對 Bug Fix、Feature、Optimization、Investigation、Refactor 及 Validation 的專屬導引，包含變更前的檢查清單與完成後的驗證標準。

## 模式說明

*   **獨立模式 (Standalone Mode)**：以目標代碼庫為唯一事實來源，建立純粹的現狀描述。
*   **參考輔助模式 (Reference-Assisted Mode)**：當專案需要參考另一個基準倉庫、API 規格書或設計文檔時使用。此模式強調「借鏡」而非「功能對齊」，除非使用者明確要求，否則參考資料僅作為架構與穩定化策略的引導。

## 安裝與開始使用

### 安裝步驟

1.  將本儲存庫複製到開發環境：
    ```bash
    git clone https://github.com/bennytsai1234/codebase-atlas.git
    ```

2.  確保你的 AI 代理人（如 Cursor, Claude Code, Windsurf 等）具備讀取檔案的權限。

### 初始化指南

請引導你的 AI 代理人執行以下指令（建議在初始化階段調用你所擁有的最強模型）：

> 讀取 `/path/to/codebase-atlas/SKILL.md`，並使用 Codebase Atlas 協議為此倉庫建立地圖。在掃描開始前，請先解決關於語言（英文或繁體中文）、模式以及交付政策的初始決策。

### 後續開發應用

地圖建立完成後，在進行具體任務時，請直接指引 AI 使用產出的文檔，例如：

> 讀取 `docs/target_bug_workflow.md`，並依照其規範修復以下問題：[問題描述]

## 技術特性

*   **語言中立**：不綁定特定的程式語言或框架，適用於任何結構化的軟體專案。
*   **純文本設計**：完全基於 Markdown 格式，無需安裝額外的資料庫、執行環境或二進位腳本，確保極高的可移植性。
*   **雙語支持**：支援繁體中文與英文輸出，並內建術語對照表以維持文檔的一致性。
*   **交付政策控制**：可針對自動化流程定義 `no commit`、`commit only` 或 `commit and push` 等交付規範。
